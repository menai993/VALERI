# VALERI synthetic seed (M1)

Deterministic synthetic dataset resembling **Ultra Higijena** (Sarajevo hygiene-products
distributor). Generated by `apps/api/valeri_api/seed/`; this folder holds the committed
ground-truth artifacts.

## How to (re)load

```bash
cd apps/api
export DATABASE_URL=postgresql+psycopg://...   # target database (migrated to head)
uv run python -m valeri_api.seed --reset       # truncate core.* and load fresh
```

Options: `--as-of YYYY-MM-DD` (reference date, default today) · `--rng-seed N`
(default 20260601) · `--manifest-out PATH` · `--export-dir PATH` (also write the
ERP-style CSV export).

## ERP-style export (the M2 ingest input)

```bash
uv run python -m valeri_api.seed --reset --export-dir /tmp/uh-export
```

Writes 4 files with Bosnian headers, `;` delimiter, UTF-8, **external codes only**
(no internal IDs, no contact PII): `kupci.csv` (sifra, naziv, jib, naziv_pravnog_lica,
segment, status, komercijalista) · `artikli.csv` (sifra, naziv, kategorija, aktivan) ·
`fakture.csv` (broj_fakture, sifra_kupca, datum, ukupno) · `stavke.csv` (broj_fakture,
sifra_artikla, kolicina, cijena, iznos). This is exactly what
`python -m valeri_api.ingest` / `POST /api/ingest/import` consumes.

The generation is **deterministic** for a given `(rng_seed, as_of)`: entity/article/customer
IDs never change; only dates shift with `as_of`.

## What is generated

| Dimension | Value |
|---|---|
| Legal entities | 72 (5 hotel groups with 2–3 objects each + 67 standalone) |
| Customers (objects) | 82 — segments: hotel (13), restoran (20), kafić (18), klinika (13), škola (16) |
| Categories | 7: papir, hemija, dispenzeri, rukavice, kozmetika, tekstil, oprema |
| Articles | 122 (120 regular + 2 created by code swaps) |
| Sales reps | 4, each owning ~20 customers |
| Invoices | ~3,400 over ~18 months, per-customer cadence 7–28 days |
| Invoice lines | ~26,000; `invoice.total` = Σ `line_total` to the cent |

## Planted cases (`planted_cases.json` = the ground truth)

These are the cases later milestones must detect — or must NOT flag. The manifest records
**measured** values from the generated data (not intentions), so M3/M4 tests can assert
against reality. IDs are stable across reseeds with the same `rng_seed`.

| Case | Count | Pattern | Must be |
|---|---|---|---|
| `declines` | 3 | last-60d turnover ≈ 40–55% of the 6-month baseline; no such drop a year ago | **detected** (M4 customer-decline) |
| `seasonal_cafes` | 2 | turnover low vs baseline, but the same 60-day window last year was equally low (yearly pattern) | **NOT flagged** (seasonal guard) |
| `lost_articles` | 4 | article bought on a regular cadence, then zero purchases for ≥3× cadence while the customer keeps ordering | **detected** (M4 lost-article) |
| `code_swaps` | 2 | old article code retired (`active=false`), `article_alias` maps old→new, purchases continue under the new code | **NOT flagged** as lost (code-swap guard) |
| `narrow_baskets` | 3 | customer buys only 1–2 categories while segment peers buy ≥4 | **detected** (M4 narrow-basket recommendation) |
| `sleeping` | 3 | ≥12 months of regular ordering, then no invoices for ≥3× the customer's average interval | **detected** (M4 sleeping-customer) |

All planted customer sets are disjoint.

## Notes

- All names, contacts, e-mails, and phone numbers are **synthetic** (no real PII).
- Money is `Decimal`/NUMERIC end-to-end — never float.
- The seed writes only to VALERI's own `core.*` schema (no source system involved).
